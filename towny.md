## /towny

| Команда | Описание |
| :--- | :--- |
| `/towny` | Показывает список базовых команд Towny. |
| `/towny ?` | Показывает расширенный список команд Towny. |
| `/towny allowedblocks` | Открывает меню с блоками, которые разрешено ставить/ломать. |
| `/towny itemuse` | Показывает предметы из списка `item_use_ids`. |
| `/towny map` | Показывает карту Towny. |
| `/towny hud` | Включает отображение карты в интерфейсе игрока (scoreboard). |
| `/towny plotclearblocks` | Показывает блоки, удаляемые командой `/plot clear`. |
| `/towny prices` | Показывает налоги и стоимость содержания города. |
| `/towny switches` | Показывает блоки из списка `switch_ids`. |
| `/towny time` | Показывает время до следующего сбора налогов/обслуживания. |
| `/towny top residents {all/town/nation}` | Показывает топ жителей (по всем, по городу, по нации). |
| `/towny top land {all/resident/town}` | Показывает топ землевладельцев (все, резиденты, города). |
| `/towny spy` | Команда администратора для слежки за всеми чатами. |
| `/towny tree` | Показывает разнообразную техническую информацию. |
| `/towny universe` | Показывает полную статистику Towny (жители, города, нации, миры, занятые участки). |
| `/towny v` | Показывает версию Towny. |
| `/towny wildsblocks` | Показывает блоки, которые можно использовать на диких участках и в дикой природе. |

## /plot

### Основные команды

| Команда | Описание |
| :--- | :--- |
| `/plot` | Показывает список команд `/plot`. |
| `/plot claim` | Приватизировать участок, выставленный на продажу (команда резидента). |
| `/plot auto` | Автоматически приватизировать несколько участков вокруг (если они продаются). |
| `/plot unclaim` | Отказаться от личного участка (резидент). |
| `/plot unclaim circle {радиус}` | Отказаться от личных участков в форме круга (радиус от текущей позиции). |
| `/plot unclaim rect {радиус}` | Отказаться от личных участков в форме прямоугольника. |
| `/plot forsale` / `fs {цена}` | Выставить участок на продажу за указанную цену. |
| `/plot forsale circle {радиус} {цена}` | Выставить на продажу участки в форме круга. |
| `/plot forsale rect {радиус} {цена}` | Выставить на продажу участки в форме прямоугольника. |
| `/plot notforsale` / `nfs` | Снять участок с продажи. |
| `/plot nfs circle {радиус}` | Снять с продажи участки в форме круга. |
| `/plot nfs rect {радиус}` | Снять с продажи участки в форме прямоугольника. |
| `/plot evict` | Выселить владельца с участка (обычно используется мэром). |
| `/plot evict forsale` | Выселить владельца и сразу выставить участок на продажу. |

### Работа с районами (`/plot district`)

| Команда | Описание |
| :--- | :--- |
| `/plot district create` / `new [имя]` | Создать новый район с указанным именем. |
| `/plot district add [имя]` | Добавить текущий участок в район. Если активно добавление, имя можно не повторять. |
| `/plot district rename [имя]` | Переименовать район. |
| `/plot district remove` | Убрать текущий участок из района. |
| `/plot district delete` | Удалить район, в котором вы стоите. |

### Доверие и права на участке (`/plot trust`, `/plot perm`)

| Команда | Описание |
| :--- | :--- |
| `/plot trust add {игрок}` | Добавить игрока в доверенные на этом участке. |
| `/plot trust remove {игрок}` | Убрать игрока из доверенных на этом участке. |
| `/plot perm` | Показать строку прав (`perm`) участка, на котором стоите. |
| `/plot perm gui` | Открыть графический интерфейс настройки персональных прав участка. |
| `/plot perm add {игрок}` | Добавить игрока в список персонального переопределения прав на участке. |
| `/plot perm remove {игрок}` | Убрать игрока из списка переопределения прав. |
| `/plot perm hud` | Включить/выключить отображение строки прав на экране (scoreboard). |

### Установка типа участка (`/plot set`)

| Команда | Описание |
| :--- | :--- |
| `/plot set reset` | Вернуть тип участка к обычному (сбрасывает shop/embassy/arena/wilds). |
| `/plot set shop` | Установить участок как магазин. |
| `/plot set embassy` | Установить участок как посольство. |
| `/plot set arena` | Установить участок как арену. |
| `/plot set wilds` | Установить участок как дикий (wilds). |
| `/plot set inn` | Установить участок как гостиницу. |
| `/plot set jail` | Установить участок как тюрьму. |
| `/plot set farm` | Установить участок как ферму. |
| `/plot set bank` | Установить участок как банк. |
| `/plot set outpost` | Установить участок как форпост (стоимость как у `/t claim outpost`). |
| `/plot set spawn` | Установить точку спавна внутри существующего форпоста на ваше текущее место. |
| `/plot set name` | Переименовать участок (мэр или владелец). Заменяет стандартную надпись «~Unowned». |

### Права на участке (`/plot set perm`)

| Команда | Описание |
| :--- | :--- |
| `/plot set perm on/off` | Включить/выключить переопределение прав на конкретном участке. |
| `/plot set perm {resident/ally/outsider} on/off` | Включить/выключить права для всех действий у указанной группы. |
| `/plot set perm {build/destroy/switch/itemuse} on/off` | Включить/выключить конкретное действие для всех групп. |
| `/plot set perm {resident/ally/outsider} {build/destroy/switch/itemuse} on/off` | Точечная настройка конкретного действия для конкретной группы. |
| `/plot set perm reset` | Сбросить права участка к стандартным (из `/town` или `/resident`). |

### Ограничения по дням (`/plot set minjoindays`, `/plot set maxjoindays`)

| Команда | Описание |
| :--- | :--- |
| `/plot set minjoindays {число}` | Установить минимальное количество дней в городе для привата участка. |
| `/plot set minjoindays clear` | Убрать ограничение минимальных дней. |
| `/plot set maxjoindays {число}` | Установить максимальное количество дней в городе для привата участка. |
| `/plot set maxjoindays clear` | Убрать ограничение максимальных дней. |

### Переключатели участка (`/plot toggle`)

| Команда | Описание |
| :--- | :--- |
| `/plot toggle fire` | Вкл/выкл распространение огня на участке. |
| `/plot toggle pvp` | Вкл/выкл PvP на участке. |
| `/plot toggle explosion` | Вкл/выкл взрывы на участке. |
| `/plot toggle mob` | Вкл/выкл появление враждебных мобов на участке. |
| `/plot toggle taxed` | Вкл/выкл налог для личного владельца участка. |

### Очистка участка (`/plot clear`)

| Команда | Описание |
| :--- | :--- |
| `/plot clear` | Удалить блоки из списка `plotclearblocks` на участке (мэр на городской земле, или владелец на личном). |

### Работа с группами участков (`/plot group`)

| Команда | Описание |
| :--- | :--- |
| `/plot group add/new/create {имя}` | Создать группу участков или добавить текущий участок в существующую группу. |
| `/plot group remove` | Убрать текущий участок из группы. |
| `/plot group delete` | Полностью удалить группу участков. |
| `/plot group rename {новое_имя}` | Переименовать группу участков. |
| `/plot group set {тип}` | Установить тип для всей группы (кроме Jail). |
| `/plot group set perm ...` | Настроить права всей группы (синтаксис как у `/plot set perm`). |
| `/plot group toggle ...` | Применить переключатели ко всей группе (как `/plot toggle`). |
| `/plot group forsale/fs {цена}` | Выставить всю группу на продажу. |
| `/plot group notforsale/nfs` | Снять группу с продажи. |

#### Доверие и права внутри группы (`/plot group trust`, `/plot group perm`)

| Команда | Описание |
| :--- | :--- |
| `/plot group trust add {игрок}` | Добавить игрока в доверенные для всей группы. |
| `/plot group trust remove {игрок}` | Убрать игрока из доверенных группы. |
| `/plot group perm gui` | Открыть GUI для настройки персональных прав группы. |
| `/plot group perm add {игрок}` | Добавить игрока в переопределение прав группы. |
| `/plot group perm remove {игрок}` | Убрать игрока из переопределения прав группы. |

### Тюремные камеры (`/plot jailcell`)

| Команда | Описание |
| :--- | :--- |
| `/plot jailcell add` | Добавить тюремную камеру на тюремном участке в месте, где вы стоите. |
| `/plot jailcell remove` | Удалить тюремную камеру с тюремного участка. |

## /resident

### Основные команды

| Команда | Описание |
| :--- | :--- |
| `/resident` | Показывает экран резидента. |
| `/resident ?` | Показывает доступные команды `/res`. |
| `/resident {резидент}` | Показывает экран другого резидента. |

### Друзья (`/resident friend`)

| Команда | Описание |
| :--- | :--- |
| `/resident friend add {игрок} .. {игрок}` | Добавить онлайн-игрока(ов) в список друзей. |
| `/resident friend add+ {игрок} .. {игрок}` | Добавить офлайн-игрока(ов) в список друзей. |
| `/resident friend remove {игрок} .. {игрок}` | Удалить онлайн-игрока(ов) из друзей. |
| `/resident friend remove+ {игрок} .. {игрок}` | Удалить офлайн-игрока(ов) из друзей. |
| `/resident friend clearlist` | Удалить всех друзей из списка. |
| `/resident friend list` | Показать список ваших друзей. |
| `/resident friend list online` | Показать список друзей, которые сейчас онлайн. |

### Список резидентов, тюрьма и вне закона

| Команда | Описание |
| :--- | :--- |
| `/resident list` | Показать список резидентов из папки данных Towny, которые онлайн. |
| `/resident jail paybail` | Заплатить штраф за выход из тюрьмы. Деньги получает город, владеющий тюрьмой. |
| `/resident outlawlist [игрок]` | Показать список городов, где указанный игрок объявлен вне закона. Без имени — свои города. |

### Участки резидента (`/resident plotlist`)

| Команда | Описание |
| :--- | :--- |
| `/resident plotlist [страница]` | Показать список своих личных участков. Можно указать номер страницы. |
| `/resident plotlist [имя] [страница]` | Показать список участков указанного игрока. |

### Спавн (`/resident spawn`)

| Команда | Описание |
| :--- | :--- |
| `/resident spawn` | Телепортирует к своей кровати (если `deny_bed_use: true` и у игрока установлен спавн в кровати). |

### Переключатели (`/resident toggle`)

| Команда | Описание |
| :--- | :--- |
| `/resident toggle pvp` | Вкл/выкл PvP на всех личных участках резидента. |
| `/resident toggle fire` | Вкл/выкл распространение огня на всех личных участках. |
| `/resident toggle explosion` | Вкл/выкл взрывы на всех личных участках. |
| `/resident toggle mobs` | Вкл/выкл враждебных мобов на всех личных участках. |
| `/resident toggle map` | Включает карту Towny, обновляющуюся при пересечении границ участков. |
| `/resident toggle townclaim` | Режим авто-привата: при ходьбе автоматически выполняется `/t claim`. |
| `/resident toggle townunclaim` | Режим авто-отказа: при ходьбе автоматически выполняется `/t unclaim`. |
| `/resident toggle bedspawn` | Режим предпочтения спавна у кровати, а не в городе. |
| `/resident toggle plotborder` | Вкл/выкл дымчатую границу участка при переходе между чанками. |
| `/resident toggle constantplotborder` | Вкл постоянную дымчатую границу участков (не исчезает). |
| `/resident toggle townborder` | Вкл постоянную дымчатую границу того города, в котором стоите. |
| `/resident toggle bordertitles` | Вкл/выкл сообщения-заголовки при входе/выходе из города (если `using_titles: true` в конфиге). |
| `/resident toggle plotgroup` | Включает режим автоматического добавления участков в строящуюся группу участков при заходе на них. |
| `/resident toggle district` | Включает режим автоматического добавления участков в строящийся район при заходе на них. |
| `/resident toggle spy` | Для администраторов: включить слежку за всеми чат-каналами. |
| `/resident toggle ignoreinvites` | Вкл/выкл игнорирование приглашений в города. |
| `/resident toggle ignoreplots` | Вкл/выкл игнорирование уведомлений об участках в городе. |
| `/resident toggle reset` | Выключает все активные режимы. |
| `/resident toggle infotool` | Для администраторов: позволяет использовать глиняный кирпич для получения Bukkit-имён блоков и сущностей. |
| `/resident toggle adminbypass` | Для администраторов: игнорировать ограничения разрешений (режим «как не админ»). |

### Настройка прав (`/resident set perm`)

| Команда | Описание |
| :--- | :--- |
| `/resident set perm on/off` | Включить/выключить переопределение прав на экране резидента. |
| `/resident set perm {friend/ally/outsider} on/off` | Вкл/выкл все права для указанной группы. |
| `/resident set perm {build/destroy/switch/itemuse} on/off` | Вкл/выкл конкретное действие для всех. |
| `/resident set perm {friend/ally/outsider} {build/destroy/switch/itemuse} on/off` | Точечная настройка конкретного действия для конкретной группы. |
| `/resident set perm reset` | Применить стандартную строку прав (из `/resident`) ко всем своим личным участкам. |

### Статус «О себе» (`/resident about`)

| Команда | Описание |
| :--- | :--- |
| `/resident about {сообщение}` | Установить текст «о себе», отображаемый в `/res`. |
| `/resident about clear` | Удаляет все ваши активные режимы. |
| `/resident about reset` | Удаляет все ваши режимы и заново присваивает стандартные. |

### Налоги (`/resident tax`)

| Команда | Описание |
| :--- | :--- |
| `/resident tax [игрок]` | Показать сумму налогов, уплачиваемых игроком (если игрок не указан — свои налоги). |

## /town

### Основные команды и информация

| Команда | Описание |
| :--- | :--- |
| `/town` | Показывает экран вашего города. |
| `/town ?` | Показывает доступные команды `/town`. |
| `/town {город}` | Показывает экран другого города. |
| `/town here` | Показывает экран города, в котором вы стоите. |
| `/town online` | Показывает жителей вашего города, которые сейчас в сети. |
| `/town plots {город}` | Показывает список участков, их типы и доход, принадлежащих городу. |
| `/town reslist {город}` | Показывает ПОЛНЫЙ список всех жителей города. |
| `/town say {сообщение}` | Отправляет сообщение всем онлайн-членам города. |
| `/town allylist {город}` | Показывает список союзников города. |
| `/town enemylist {город}` | Показывает список врагов города. |
| `/town nearby` | Показывает до 10 ближайших городов. |
| `/town leave` | Покинуть город. |
| `/town reclaim` | Позволяет жителю восстановить свой разрушенный город. |
| `/town merge {город}` | Присоединить указанный город к вашему (указанный город вливается в ваш). |
| `/town delete {город}` | Команда админа/мэра для удаления города из файлов Towny. |

### Создание города и управление жителями

| Команда | Описание |
| :--- | :--- |
| `/town new {название}` | Создать новый город. |
| `/town add {житель} .. {житель}` | Мэр добавляет жителей в город. |
| `/town kick {житель} .. {житель}` | Мэр выгоняет жителей из города. |
| `/town join {город}` | Вступить в открытый город (не требующий приглашений). |
| `/town purge {дни}` | Выгнать жителей, неактивных указанное количество дней (кроме мэров и NPC). |

### Спавн

| Команда | Описание |
| :--- | :--- |
| `/town spawn` | Телепортирует на спавн вашего города. |
| `/town spawn {город}` | Телепортирует на спавн другого города. |

### Список городов (`/town list`)

| Команда | Описание |
| :--- | :--- |
| `/town list by name {стр}` | Сортировка по алфавиту. |
| `/town list by resident {стр}` | Сортировка по количеству жителей (больше — выше). |
| `/town list by balance {стр}` | Сортировка по балансу банка нации. |
| `/town list by townblocks {стр}` | Сортировка по количеству захваченных участков. |
| `/town list by forsale {стр}` | Сортировка по выставленным на продажу городам, по цене. |
| `/town list by online {стр}` | Сортировка по количеству игроков онлайн. |
| `/town list by open {стр}` | Сначала открытые города, затем по убыванию жителей. |
| `/town list by public {стр}` | Сначала публичные города, затем по убыванию жителей. |
| `/town list by ruined {стр}` | Сначала разрушенные города, затем по убыванию жителей. |
| `/town list by bankrupt {стр}` | Сначала обанкротившиеся города, затем по убыванию жителей. |
| `/town list by founded {стр}` | Сортировка по дате основания, старые сначала. |

### Управление участками

| Команда | Описание |
| :--- | :--- |
| `/town claim` | Приватизировать участок, на котором стоите, для города. |
| `/town claim outpost <#\|{имя}\|{имя:#}>` | Приватизировать форпост. `{имя}` — по названию, `{имя:#}` — если имя начинается с цифры. |

| **Покупка участков** | Могут купить участок посольства, если имеют право `towny.command.plot.claim`. |
| **Автоматические права** | Все кочевники получают ноду `towny.townless`. |

### 🏠 Жители (Residents)

| Характеристика | Описание |
| :--- | :--- |
| **Кто это** | Игрок, вступивший в город (или автоматически зачисленный при первом входе). |
| **По умолчанию** | При первом заходе получают ноду `towny.town.resident` (прописана в секции nomad файла `townyperms.yml`). |
| **Команда** | `/resident` — открывает экран резидента: баланс, город, участки, друзья. |
| **Авто-город** | Можно задать `default_town_name: 'Название'` в `config.yml`, чтобы новые игроки сразу попадали в конкретный город. |
| **Участки** | Могут приватить участки, выставленные мэром на продажу. При наличии ≥1 участка в `/res` появляется строка с количеством участков и стандартными правами на них (если не заданы индивидуальные). |
| **Права** | Настраиваются в секции `towns.default` файла `TownyPerms.yml`. |

### 🏙️ Города (Towns)

| Характеристика | Описание |
| :--- | :--- |
| **Что это** | Объединение жителей (минимум один) с одним мэром. |
| **Банк** | Город имеет банк, из которого мэр может снимать деньги. |
| **Помощники** | Мэр может назначать ассистентов с аналогичными или ограниченными правами. |
| **Налоги** | Взимаются с жителей в конце каждого «дня» (интервала). |
| **Рост** | Города расширяются от домашнего блока (`homeblock`), захватывая соседние участки, либо создавая аванпосты (`/t claim outpost`). |
| **Ограничения** | Максимальное число жителей можно установить в `global_town_settings.max_residents_per_town` (по умолчанию без ограничений). |
| **Список** | `/town list` показывает все города сервера. |
| **Метаданные** | С версии 0.95.1.0 можно хранить метаданные города. |
| **Продажа города** | С версии 0.99.6.0 город можно выставить на продажу (`/t forsale [$]`), купить (`/t buytown [город]`) или снять с продажи (`/t notforsale`). |

### 👑 Мэры (Mayors)

| Характеристика | Описание |
| :--- | :--- |
| **Кто это** | Лидер города, управляет им вместе с помощниками. |
| **Права** | Настраиваются через `TownyPerms.yml`. |
| **Ранги** | Мэр может назначать жителям ранги (помощник, казначей, строитель и т.д.), созданные админом в `townyperms.yml`. |
| **Назначение рангов** | `/town rank {add\|remove} {игрок} {ранг}`. Один игрок может иметь несколько рангов. |
| **Налоги** | Мэр определяет тип и ставку налогов для города. |

#### 👥 Несколько городов под управлением админа (NPC-мэры)

Администратор может управлять несколькими городами одновременно, используя NPC-мэра.

**Пример: Админ Боб и NPC-город**

- Боб создаёт «Серверный Город», настраивает налоги, цены на участки, права.
- Выдаёт городу неограниченные участки: `/ta town {город} toggle unlimitedclaims`
- Назначает NPC-мэром: `/townyadmin set mayor {город} npc`
- Покидает город: `/t leave`
- Создаёт свой личный город: `/t new {название}`

**Дальнейшее управление:**

- Боб может добавлять игроков в Серверный Город, не покидая свой: `/ta town {город} add {игрок}`, или прописать `default_town_name` в конфиге.
- Настройки города меняются через `/ta town {город} ...`
- NPC-город можно добавить в нацию: `/ta nation {нация} add {город}`
- Создать сразу NPC-город, не выходя из своего: `/ta town new {город} npc`

#### 🏷️ Титулы и суффиксы (с версии 0.95.1.0)

| Команда | Описание |
| :--- | :--- |
| `/town set title {имя} текст` | Добавить титул (префикс) жителю. |
| `/town set surname {имя} текст` | Добавить суффикс (постфикс) жителю. |

### 🚫 Преступники (Outlaws)

| Характеристика | Описание |
| :--- | :--- |
| **Что это** | Список игроков, объявленных вне закона в конкретном городе. |
| **Назначение** | `/town outlaw [add\|remove] [имя]` (требует право `towny.command.town.outlaw`). |
| **Кто может быть** | Любой игрок, не обязательно член города или нации. |
| **Последствия** | — Если преступник был жителем города, его немедленно выгоняют.<br>— В открытый город (`Open`) преступники не могут вступить через `/t join`.<br>— Не могут телепортироваться на спавн публичного города.<br>— При входе в город видят предупреждение в виде заголовка.<br>— Если игрок в сети и его объявили вне закона, он увидит сообщение в чате. |
| **Тюрьма за смерть** | Если преступник умирает в городе, где он вне закона, его могут посадить в тюрьму, если:<br>- `jail.is_jailing_attacking_outlaws: true` в конфиге<br>- убивший имеет право `towny.outlaw.jailer`<br>- в городе есть тюремный участок<br>По умолчанию право `towny.outlaw.jailer` есть у мэра, ассистента и шерифа. |
| **Просмотр списка** | `/town outlawlist` — для своего города.<br>`/town outlawlist {город}` — для любого города.<br>`/resident outlawlist [игрок]` — список городов, где игрок (или вы) вне закона. |
| **Телепортация** (с 0.96.4.0) | Если `allow_outlaws_to_enter_town: false`, преступники телепортируются прочь. Задержка задаётся в `outlaw_teleport_warmup` (0 — мгновенно). |
| **Принудительное PvP** (с 0.101.2.0) | — `force_pvp_on_outlaws: true` — преступников всегда можно атаковать.<br>- `allows_outlaws_to_always_pvp: true` — преступники сами могут атаковать в городах, где они вне закона. |

## 🚩 Нации (Nations)

### Общая информация

| Характеристика | Описание |
| :--- | :--- |
| **Что это** | Объединение городов (минимум один) с одним городом-столицей. |
| **Лидер** | Мэр столицы автоматически становится лидером нации. |
| **Банк** | Нация имеет собственный банк. |
| **Налоги** | Может облагать налогом входящие в неё города. |
| **Война** | Может участвовать в военных событиях и заключать альянсы. |

### Спавн и публичность

| Особенность | Описание |
| :--- | :--- |
| **Спавн нации** | `/nation spawn` — телепортирует на точку спавна нации. |
| **Публичный спавн** | Если нация публичная, на её спавн могут телепортироваться почти все игроки, кроме врагов. |
| **Ограничение спавна** | Можно привязать спавн строго к столице (`global_nation_settings.capital_spawn`), иначе точка может быть где угодно. |

### Альянсы

| Характеристика | Описание |
| :--- | :--- |
| **Что дают** | Защита от friendly-fire, возможность помогать на участках (если разрешено союзникам), совместные военные действия. |
| **Двусторонние альянсы** | С версии 0.91.0.0 можно запретить односторонние альянсы: `war.disallow_one_way_alliance: true` (по умолчанию `false`). |

### Ограничения и требования

| Версия | Настройка | Описание |
| :--- | :--- | :--- |
| 0.91.0.0 | `global_town_settings.minimum_residents_for_nation` | Минимальное число жителей для создания/поддержания нации. |
| 0.96.2.0 | `global_town_settings.maximum_number_residents_without_nation` | Максимум жителей в городе без нации. При >0 городам придётся создавать нации для роста. |

### NationZone (Национальная зона)

| Характеристика | Описание |
| :--- | :--- |
| **Что это** | Зона вокруг городов нации, где строить/ломать могут только члены нации (как обычная дикая местность, но с защитой). |
| **Включение** | `global_nation_settings.nationzone.enable: true` |
| **Размер** | Увеличивается с ростом населения через `NationLevels` в конфиге. Можно сделать зону больше у столицы. |
| **Отключение во время войны** | `global_nation_settings.nationzone.war_disables` |
| **Ручное управление** | Мэр может выключить зону города: `/t toggle nationzone`. Админ может задать размер: `/ta town set nationzoneoverride #`. |

### Расстояния и санкции (с 0.100.1.0)

| Особенность | Описание |
| :--- | :--- |
| **Максимальное расстояние** | Можно ограничить расстояние между столицей и городами, а также между городами внутри нации (см. `proximity` в `Global Nation Settings`). |
| **Санкции городов** | `/nation sanctiontown` — запретить городу вступать в нацию через `/n join` и использовать `/n spawn [нация]` (если нация публичная). |

### Привилегии от роста населения

- Бонусные участки для городов.
- Снижение стоимости содержания города.
- Увеличенная NationZone.
- Повышенный лимит аванпостов.

## 📊 Настройка уровней города и нации

### 🏙️ Уровень города (`town_level`)

Пример структуры уровня города:

```
  - numResidents: 1
    namePrefix: ''
    namePostfix: ' (Settlement)'
    mayorPrefix: 'Hermit '
    mayorPostfix: ''
    townBlockBuyBonusLimit: 0
    townBlockLimit: 16
    upkeepModifier: 1.0
    townOutpostLimit: 0
    debtCapModifier: 1.0
    peacefulCostMultiplier: 1.0
    bankCapModifier: 1.0
    resourceProductionModifier: 1.0
    townBlockTypeLimits:
    - shop: 2
    - arena: 2

  - numResidents: 2
    namePrefix: ''
    namePostfix: ' (Hamlet)'
    mayorPrefix: 'Chief '
    mayorPostfix: ''
    townBlockBuyBonusLimit: 0
    townBlockLimit: 32
    upkeepModifier: 1.0
    townOutpostLimit: 1
    debtCapModifier: 1.0
    peacefulCostMultiplier: 1.0
    bankCapModifier: 1.0
    resourceProductionModifier: 1.0
    townBlockTypeLimits:
    - shop: 5
    - arena: 4
```

### Пояснение переменных `town_level`

| Переменная | Описание |
| :--- | :--- |
| `numResidents: 1` | Количество жителей, необходимое городу для достижения данного уровня `town_level`. Может определяться количеством TownBlocks, если `global_town_settings.town_level_is_determined_by_townblock_count_instead_of_resident_count: true`. |
| `namePrefix: ''` | Добавляется **перед** названием города. |
| `namePostfix: ' (Settlement)'` | Добавляется **после** названия города. |
| `mayorPrefix: 'Hermit '` | Добавляется **перед** именем мэра. |
| `mayorPostfix: ''` | Добавляется **после** имени мэра. |
| `townBlockBuyBonusLimit: 0` | Максимальное количество бонусных участков, которые город может купить через `/town buy bonus`. Требует `town.max_purchased_blocks_uses_town_levels: true`. |
| `townBlockLimit: 16` | Переопределяет `town_block_ratio` и задаёт точное количество городских участков. Требует `town_block_ratio: '0'`. |
| `upkeepModifier: 1.0` | Множитель стоимости содержания города. При `town_plotbased_upkeep: true` влияет только если `town_plotbased_upkeep_affected_by_town_level_modifier: true`. |
| `townOutpostLimit: 1` | Лимит аванпостов. Требует `limit_outposts_using_town_and_nation_levels: true`. |
| `debtCapModifier: 1.0` | Множитель лимита задолженности. При `debt_cap_uses_town_levels: true` финальный лимит = `debt_cap.override` × `debtCapModifier`. |
| `peacefulCostMultiplier: 1.0` | Влияет на стоимость нейтралитета города (цена населения × `economy.price_town_neutrality`). |
| `resourceProductionModifier: 1.0` | Изменяет количество ресурсов, ежедневно получаемых городом (плагин TownyResources). |
| `bankCapModifier: 1.0` | Множитель лимита банка города (население × `economy.banks.town_bank_cap`). |
| `townBlockTypeLimits` | Список ограничений на количество участков каждого типа (например, `shop: 2`). Может быть пустым `[]`. |

👉 В примере: при 1 жителе город получает суффикс `(Settlement)`, мэр — префикс `Hermit`. При 2 жителях — `(Hamlet)` и префикс `Chief`.

## 🏗️ Как растут города (How Towns Grow)

### 🚀 Создание города (Starting a Town)

| Характеристика | Описание |
| :--- | :--- |
| **Команда** | `/town new {название}` — создаёт новый город. |
| **Стоимость** | Обычно требует сумму, заданную в `price_new_town`. |
| **Домашний блок** | Городской блок, на котором стоял мэр при создании, становится `homeblock`. Точная позиция — спавн города. |
| **Смена спавна** | `/t set spawn` — переносит точку спавна в пределах домашнего блока. |
| **Смена домашнего блока** | `/t set homeblock` — переносит домашний блок на другой захваченный участок. |
| **Захват участков** | `/town claim` — новые участки должны примыкать к уже захваченным, кроме аванпостов. |
| **Бонус новым городам** | С версии 0.100.2.0 можно выдать разовый бонус участков: `claiming.new_town_bonus_claims` в конфиге. |

### 📥 Вступление в город (Joining Towns)

| Способ | Описание |
| :--- | :--- |
| **По приглашению** | Мэр или ассистент вводит `/town add {игрок}`. Игрок получает запрос и может `/accept` или `/deny`. |
| **Открытый город** | Мэр включает `/town toggle open`. Игрок без города может войти командой `/town join {город}`. Список открытых городов: `/town list by open`. |
| **Эффект** | Новые жители увеличивают лимит участков, доступных мэру для захвата. |

## 📐 Система владения участками (Plot System of Land Ownership)

### 🧱 Городские блоки (Town Blocks)

| Характеристика | Описание |
| :--- | :--- |
| **Концепция** | Защита блоков разбита на участки — townblocks. По умолчанию 16×16×N (N — высота мира). Это равномерная сетка без промежутков. |
| **Захват** | Мэры забирают городские блоки, затем продают/передают их жителям. |
| **Метаданные** | С версии 0.95.1.0 можно хранить метаданные на участке. |

### 📏 Размер городского блока (Town Block Size)

| Настройка | Описание |
| :--- | :--- |
| `town_block_size: 16` | Меняется в `config.yml`. Рекомендуется задавать только при первой установке Towny. Изменение позже сдвинет всё нежелательно. |
| **Меньший размер** | Даёт больше точности, но больше работы. Не рекомендуется меньше 4 (иначе страдает кэширование). |

### 🏁 Захват городских блоков (Claiming Townblocks)

#### Захват для города (Town Claiming)

| Особенность | Описание |
| :--- | :--- |
| **Кто может** | По умолчанию — мэр и ассистенты, имеющие соответствующие права. |
| **Массовый захват** | Требует право `towny.command.town.claim.town.multiple`. |

##### Команды захвата

| Команда | Результат |
| :--- | :--- |
| `/town claim` | Захватывает один участок под игроком. |
| `/town claim #` | Квадрат с радиусом `#` (может немного уменьшиться до идеального квадрата). |
| `/town claim rect #` | Квадрат с радиусом `#`. |
| `/town claim rect auto` | Захватывает максимально возможный квадрат вокруг игрока. |
| `/town claim circle #` | Круг с радиусом `#`. |
| `/town claim circle auto` | Максимальный круг вокруг игрока. |
| `/town claim auto` | Максимальный квадрат вокруг игрока. |
| `/town claim fill` | Заполняет внутреннюю область, окружённую границей из захваченных участков. Сначала нужна замкнутая граница. |

#### Прочие механики

| Механика | Описание |
| :--- | :--- |
| **Доступные участки** | `/town` показывает, сколько участков доступно для захвата. |
| **Возврат за снятие захвата** (с 0.95.0.0) | `economy.new_expand.price_claim_townblock_refund`. Не рекомендуется ставить ≥ стоимости захвата. С 0.98.0.0 можно задать отрицательное число — плата за отказ от участка. |
| **Рост стоимости захвата** (с 0.95.0.0) | `price_claim_townblock_increase`. 1 — отключено. 1.3 означает +30% к каждому последующему участку. Текущую стоимость можно увидеть в `/towny prices`. |
| **Максимальная цена** (с 0.96.3.0) | `economy.new_expand.max_price_claim_townblock` ограничивает рост. |
| **Форма захвата** (с 0.99.0.0) | `claiming.min_adjacent_blocks` — требует указанное количество соседних участков для захвата (2 или 3), чтобы запретить захват в виде линии. -1 — отключено. |
| **Запрет биомов** (с 0.100.1.0) | В секции `claiming.biome_rules` можно запретить захват в океанах и нежелательных биомах, если процент биома на участке превышает лимит. |

| :--- | :--- |

| Житель приватирует участок | `/plot claim`. |
| Житель отказывается от участка | `/plot unclaim`. |

| Источник | Настройка | Поведение |
| :--- | :--- | :--- |
| **Основной параметр** | `town_block_ratio: 8` | Город получает 8 блоков на жителя. |
| **Через уровни** | `town_block_ratio: 0` + секция `town_level` | Количество блоков задаётся индивидуально для каждого уровня города. |

| **Неограниченные блоки** (с 0.98.0.0) | `town_block_ratio: -1` (всем городам) или `/ta town ГОРОД toggle unlimitedclaims` (одному) | Снимает лимит блоков. |

## 🔄 Захват чужих блоков (Overclaiming Townblocks)

| Характеристика | Описание |
| :--- | :--- |
| **Включение** | `town.overclaiming.being_overclaimed_allows_other_towns_to_steal_land: true` (по умолчанию отключено). |
| **Условие для города-жертвы** | Город считается «overclaimed», когда его текущее число блоков превышает лимит (например, 120/100). |
| **Команда захвата** | `/t takeoverclaim` — забрать один соседний блок у перегруженного города. |
| **Требования** | — Использовать на границе своего и чужого города.<br>— Ваш город не должен быть перегружен.<br>— Хватает денег для оплаты (`economy.takeoverclaim.price`).<br>— Возможно, требоваться минимальный возраст города.<br>— Может быть кулдаун. |
| **Уведомления** | Мэр перегруженного города видит предупреждения при входе, при подходе к лимиту, при потере жителей. На доступных для захвата блоках отображается `[TakeoverClaim Available]`. |
| **Ограничения** | Нельзя разделить город на две части — захват идёт снаружи внутрь. Можно защитить радиус вокруг домашнего блока (`town.overclaiming.overclaiming_prevented_by_homeblock_radius`). |
| **Нации и враги** (с 0.100.1.0) | Опционально: захват блоков нации разрешён только врагам (`claiming.overclaiming.nations_required_to_be_enemies`). |

> **Рекомендация:** Лучше всего использовать на серверах с отключёнными аванпостами и требованием более 1 соседнего блока для захвата.

## 📦 Группы участков (Plot Groups)

| Характеристика | Описание |
| :--- | :--- |
| **Что это** | Несколько участков, объединённых в группу для продажи, покупки и управления как единым целым. |
| **Создание/добавление** | Игрок стоит на участке и вводит `/plot group add {имя_группы}`. Если группа с таким именем существует, участок добавится в неё. Если нет — создастся новая. |

| **Команды управления** | Префикс меняется на `/plot group` (например, `/plot group fs 10000`, `/plot group set perm ...`). Полный список: `/plot group ?`. |
| **Авторежим добавления** (с 0.99.6.0) | — Включить: `/res toggle plotgroup` после создания группы.<br>— При ходьбе каждый участок, на который заходит игрок, автоматически добавляется в группу.<br>— Выключить: повторный `/res toggle plotgroup`. |

## 🛠️ Пользовательские типы участков (Custom Plot Types)

Пользовательский тип участка состоит из 7 элементов:

| Ключ | Описание |
| :--- | :--- |
| `name` | Название, используемое для этого типа участка в игре и базе данных. |
| `cost` | Стоимость, которую игрок платит за установку участку данного типа. |
| `tax` | Налог, взимаемый с владельца каждый день. Если `tax: 0`, используется обычный налог на участок (`plottax`). |
| `mapKey` | Символ, отображаемый на карте `/towny map`. |
| `itemUseIds` | Если пусто, используются значения из `protection.item_use_ids`. Иначе задаёт список предметов, считающихся действием `item_use` на этом типе участка. |
| `switchIds` | Если пусто, используются значения из `protection.switch_ids`. Иначе задаёт список блоков, считающихся действием `switch` на этом типе участка. |
| `allowedBlocks` | Список блоков, которые можно ломать/ставить, если у игрока есть права build/destroy. Если пусто — действуют обычные блоки. Пример: тип `farm`. |

## 🌿 Дикие участки (Wilds Plots)

| Характеристика | Описание |
| :--- | :--- |
| **Назначение** | `/plot set wilds` — создаёт «дикий» участок внутри города. |
| **Что можно делать** | Жителям разрешено добывать блоки из wilds ignore list (руды, деревья, цветы, грибы и т.д.). Нельзя ломать камень, землю, траву и другие ландшафтные блоки. |
| **Применение** | Создание лесных ферм, пещер, защита ландшафта с возможностью собирать ресурсы. |
| **Просмотр разрешённых блоков** | `/towny wildsblocks` — показывает, какие блоки разрешено ломать/использовать на wilds-участках. |

### Настройка Wilds Plots (краткая инструкция)

1. Откройте `towny\data\worlds\WORLDNAME.txt`.
2. Установите:
   - `unclaimedZoneBuild=false`
   - `unclaimedZoneDestroy=false`
3. Настройте `unclaimedZoneIgnoreIds`, добавив нужные блоки.
4. Разметьте wilds-участки командой `/plot set wilds`.
5. По умолчанию жители имеют права build/destroy; можно дополнительно выдать права союзникам или чужакам через `/plot set perm`.

## 🔒 Тюрьма (Jail Plots)

| Характеристика | Описание |
| :--- | :--- |
| **Назначение** | `/plot set jail` — создаёт тюремный участок. |

### Как игроки попадают в тюрьму

- **По команде мэра/шерифа**:
  - **Без залога (Bail Disabled)**:
    - `/town jail [имя]` — сажает на 1 час (только для жителей города).
    - `/town jail [имя] {часы}` — сажает на указанное количество часов.
    - `/town jail [имя] {часы} {тюрьма}` — в конкретный тюремный участок (номер).
    - `/town jail [имя] {часы} {тюрьма} {камера}` — в конкретную камеру (оба параметра — числа).
  - **С залогом (Bail Enabled)**:
    - `/town jail [имя]` — 1 час со стандартной суммой залога.
    - `/town jail [имя] {часы}` — указанные часы, стандартный залог.
    - `/town jail [имя] {часы} {залог}` — часы и сумма залога.
    - `/town jail [имя] {часы} {залог} {тюрьма}` — с указанием тюремного участка.
    - `/town jail [имя] {часы} {залог} {тюрьма} {камера}` — с указанием и участка, и камеры.
- **За смерть врага в городе**: атакующий, считающийся врагом (Enemy нации), после смерти попадает в `primaryjail` защищающегося города.
- **За смерть преступника (Outlaw)**: если `jail.is_jailing_attacking_outlaws: true`, и убивший имеет право `towny.outlaw.jailer`, преступник попадает в primaryjail.

### Освобождение из тюрьмы

- Игрок покидает город и становится кочевником.
- Мэр/шериф использует `/town unjail [имя]`.
- Заключённый платит залог: `/resident jail paybail`.
- Игрок сбегает из тюремного участка и города в дикую местность.

### Дополнительные механики тюрем

| Механика | Описание |
| :--- | :--- |
| **Запрет телепортации** | Заключённые не могут телепортироваться. |
| **Жемчуг Края/Хорус** | Запрещены, если не разрешено в конфиге. |
| **Смерть в тюрьме** | Возвращает обратно в свою тюремную камеру. |
| **Деньги за убийство** | С заключённых не выплачиваются. |
| **Статус** | Отображается в `/res [имя]` с указанием города-тюремщика. |

## 🛡️ Как Towny позволяет игрокам защищать свои блоки

### Основы системы прав

Towny автоматически защищает блоки благодаря системе участков и прав. Администратору не нужно вручную защищать землю для каждого игрока, а игроки не могут бесконтрольно захватывать территории — им нужно развивать города.

С версии **0.77.0.0** доступны **права на отдельные участки** (`per-plot permissions`).

- Когда мэр захватывает участок для города (`/town claim`), он получает права, заданные в `/town`.
- Когда житель приватирует участок (`/plot claim`), он получает права, заданные в `/res`.
- Владелец может изменить права конкретного участка: `/plot set perm`.

---

### Четыре типа прав (Perm-Types)

Каждое право можно включить или выключить для определённой группы игроков.

| Тип права | Действие |
| :--- | :--- |
| **Build** | Установка (размещение) блоков. |
| **Destroy** | Разрушение блоков, а также изменение состояния некоторых блоков (например, повторителей красного камня). |
| **Switch** | Использование «переключаемых» блоков: сундуки, шалкеровые ящики, печи, раздатчики, воронки, выбрасыватели, проигрыватели, камнерезы, столы кузнеца, лучников, коптильни, ткацкие станки, точила, компостеры, столы картографа, колокола, бочки, варочные стойки, рычаги, нажимные плиты, кнопки, деревянные двери, калитки, люки, вагонетки, магнетиты, якоря возрождения, мишени и любые другие блоки, по которым кликают. |
| **ItemUse** | Использование предметов: вагонетки, лодки, жемчуг Края, огненные шары, плоды хоруса, поводки и другие предметы, которые «применяются». |

> 💡 Списки **Switch** и **ItemUse** можно расширять группами: `BOATS`, `MINECARTS`, `WOOD_DOORS`, `PRESSURE_PLATES`, `FENCE_GATES`, `TRAPDOORS`, `SHULKER_BOXES`, `BUTTONS`.
>
> Можно добавить `SADDLE` для защиты лошадей, ослов, мулов, свиней, страйдеров от оседлывания (не рекомендуется, если не хотите запрещать пересадку на животных в городах).

Посмотреть списки блоков и предметов, используемых сервером:
- `/towny switches`
- `/towny itemuse`

Админ может проверить поведение как обычный игрок: `/res toggle adminbypass` (повторный ввод отключает режим).

---

### Группы прав (Perm-Groups)

Каждому типу права можно назначить разрешение для четырёх групп игроков.

| Группа | Обозначение в `/res` | Обозначение в `/town` | Кто входит |
| :--- | :--- | :--- | :--- |
| **Friend / Resident** | F | R | В `/res` – друзья игрока. В `/town` – жители этого города. |
| **Town / Nation** | T | N | В `/res` – жители вашего города. В `/town` – жители всех городов вашей нации. |
| **Ally** | A | A | В `/res` – жители вашей нации и союзных наций. В `/town` – только жители союзных наций. |
| **Outsider** | O | O | Все остальные игроки, не входящие в город, нацию или альянс. |

---

### Настройка прав через команды

#### 🏙️ Права городской земли (невладельческие участки)

Участки, принадлежащие городу, но не приватизированные жителями, показываются как «~Unowned». Их права задаёт мэр командой `/town set perm`.

| Команда | Результат |
| :--- | :--- |
| `/town set perm on/off` | Включает/выключает **все** права для всех типов и групп. |
| `/town set perm ally on/off` | Включает/выключает все права для союзников нации. |
| `/town set perm resident build on/off` | Включает/выключает строительство для жителей города. |

#### 🏠 Права личных участков (по умолчанию)

Когда житель приватирует участок, на него распространяются стандартные права, заданные в `/resident set perm`.

| Команда | Результат |
| :--- | :--- |
| `/resident set perm on/off` | Включает/выключает **все** права для всех типов и групп. |
| `/resident set perm friend on/off` | Включает/выключает все права для друзей. |
| `/resident set perm ally on/off` | Включает/выключает все права для союзников (вся нация + альянсы). |
| `/resident set perm outsider switch on/off` | Включает/выключает использование переключателей посторонними. |

#### 📐 Права конкретного участка

Любой личный участок можно настроить отдельно, переопределив стандартные права.

| Команда | Результат |
| :--- | :--- |
| `/plot set perm on/off` | Включает/выключает **все** права для всех на этом участке. |
| `/plot set perm friend on/off` | Все права для друзей на этом участке. |
| `/plot set perm ally on/off` | Все права для союзников на этом участке. |
| `/plot set perm outsider switch on/off` | Использование переключателей посторонними на этом участке. |

### Переключатели защиты (Toggles)

Мэры могут изменять защиту всего города:

| Команда | Действие |
| :--- | :--- |
| `/town toggle explosion` | Вкл/выкл взрывы в городе. |
| `/town toggle fire` | Вкл/выкл распространение огня. |
| `/town toggle pvp` | Вкл/выкл PvP. |
| `/town toggle mobs` | Вкл/выкл появление враждебных мобов. |

На отдельных участках (мэр или владелец):

| Команда | Действие |
| :--- | :--- |
| `/plot toggle explosion` | Вкл/выкл взрывы на участке. |
| `/plot toggle fire` | Вкл/выкл огонь на участке. |
| `/plot toggle pvp` | Вкл/выкл PvP на участке. |
| `/plot toggle mobs` | Вкл/выкл мобов на участке. |

> 🔔 **Важно:** Переключатели города (`/town toggle`) переопределяют настройки отдельных участков.  
> Команды выше меняют только стандартную строку прав в `/town` или `/res`, и **не** затрагивают участки, у которых права были изменены индивидуально.

### Распространение настроек на все участки

Чтобы применить текущие стандартные права ко **всем** участкам города или резидента:

| Команда | Результат |
| :--- | :--- |
| `/resident set perm reset` | Применяет строку прав из `/res` на все личные участки резидента. |
| `/town set perm reset` | Применяет строку прав из `/town` на все городские участки (включая переключатели). |

## ⚔️ Как Towny управляет PvP

### Уровни настройки PvP

| Уровень | Команда / Описание |
| :--- | :--- |
| **Мир** | `/townyworld toggle pvp` — включает/отключает PvP во всём мире. Если выключено, PvP невозможно даже на аренах. |
| **Принудительное PvP в мире** | `/townyworld toggle forcepvp` — принудительно включает PvP везде, игнорируя настройки городов и участков. Friendly-fire сохраняется. |

| **Friendly-fire в мире** | `/townyworld toggle friendlyfire` — включает/отключает урон по своим в мире. По умолчанию friendly-fire отключён между членами города, нации и союзниками. |
| **Город** | `/town toggle pvp` — включает/отключает PvP в городе. |
| **Участок** | `/plot toggle pvp` — управляет PvP на одном участке. |

| `plotborder` | Периодически показывает границы участка. |

## 💰 Деньги, налоги и содержание (Money, Taxes and Upkeep)

### Основы экономики

Если вы не хотите, чтобы Towny использовал экономику, установите `using_economy: false` в `config.yml`. В этом случае налоги и содержание не взимаются, а создание городов/наций и приватизация участков становятся бесплатными.

Цены настраиваются в секции `economy` файла `config.yml`. Базовые параметры находятся в `new_expand`:

| Параметр | Описание |
| :--- | :--- |
| `price_new_nation` | Стоимость создания нации. |
| `price_new_town` | Стоимость создания города. |
| `price_claim_townblock` | Базовая цена захвата городского блока (`/t claim`). |

---

### Налоги и содержание (Taxes and Upkeep)

Это две разные функции. Для их работы необходимо `using_economy: true`. Налоги и содержание взимаются одновременно каждый «день Towny» (по умолчанию 24 реальных часа, задаётся в `day_interval: 1d`). Проверить время до следующего дня можно командой `/towny time`.

Глобальное включение/отключение: `daily_taxes: true` в `config.yml`.

---

### Налоги (Taxes)

#### Налоги в городе

| Команда / Настройка | Описание |
| :--- | :--- |
| `/town set taxes {$}` | Установить налог с жителей (фиксированный или процент). |
| `/town toggle taxpercent` | Переключить между фиксированной ставкой и процентом. |
| `/town set plottax {$}` | Налог с жителя за каждый личный участок. |
| `/town set taxpercentcap {$}` | Максимальная сумма при процентном налоге (настраивается мэром). |
| `/town set shoptax {$}` | Дополнительный налог за участок-магазин (с версии 0.99.6.0). |
| `/town set embassytax {$}` | Дополнительный налог за участок-посольство. |
| `/plot toggle taxed` | Освободить конкретный участок от налога (если налог был установлен). |

**Фиксированная ставка (Flatrate):**
- Одинаковая сумма с каждого жителя.
- Если житель не может заплатить — его выгоняют из города.
- Максимальная сумма: `economy.daily_taxes.max_town_tax_amount` (по умолчанию 1000).

**Процентная ставка (Percentage):**
- Взимается процент от баланса игрока.
- Если денег нет — игрок не платит и не выгоняется.
- Можно ограничить максимальную сумму через `/town set taxpercentcap {$}`.
- Максимальный процент: `economy.daily_taxes.max_town_tax_percent` (по умолчанию 25%).

**Налог на участки (Plottax):**
- Взимается с каждого личного участка жителя.
- Пример: при `plottax = 10` владелец 4 участков заплатит 40.
- Если житель не может заплатить — теряет участок.
- Максимум: `economy.daily_taxes.max_plot_tax_amount`.

#### Налоги в нации

| Команда / Настройка | Описание |
| :--- | :--- |
| `/nation set taxes {$}` | Установить налог с городов (фиксированный или процент). |
| `/nation toggle taxpercent` | Переключить между фиксированной ставкой и процентом. |
| `/nation set taxpercentcap {$}` | Максимальная сумма при процентном налоге. |
| `/nation set conqueredtax {$}` | Налог на захваченные города (со статусом Conquered). |

**Фиксированная ставка:**
- Одинаковая сумма с каждого города нации.
- Если город не платит — исключается из нации.
- Максимум: `economy.daily_taxes.max_nation_tax_amount`.

**Процентная ставка:**
- Процент от банка города.
- Если банк пуст — город не платит и не исключается.
- Максимальный процент: `economy.daily_taxes.max_nation_tax_percent`.

**Налог на захваченные города (`conqueredtax`):**
- Взимается ежедневно с городов, проигравших войну.
- При неуплате может наступить банкротство (если включено).
- Настройки по умолчанию: `nation.default_taxes.default_nation_conquered_tax` и `max_nation_conquered_tax`.

#### Отрицательный налог (с 0.99.6.0)

| Параметр в `config.yml` | Описание |
| :--- | :--- |
| `economy.daily_taxes.allow_negative_town_tax: true` | Разрешить городу устанавливать налог ниже нуля — город платит жителям. |
| `economy.daily_taxes.allow_negative_nation_tax: true` | Разрешить нации устанавливать налог ниже нуля — нация платит городам. |

#### Выплаты землевладельцам

| Действие | Описание |
| :--- | :--- |
| Установить отрицательный `price_town_upkeep` и `use_plot_payments: true` | Каждый день сумма содержания (отрицательная) распределяется между владельцами личных участков. Выплаты идут от сервера, а не из банка города. |

---

### Содержание (Upkeep)

Содержание списывается из банков городов и наций и изымается из игры. Можно установить отрицательное значение — тогда сервер будет **платить** городам и нациям. Помогает удалять неактивные города и контролировать инфляцию.

| Параметр | Описание |
| :--- | :--- |
| `price_town_upkeep: 10.0` | Ежедневная плата с каждого города. При неуплате все жители выгоняются, город удаляется. |
| `price_nation_upkeep: 100.0` | Ежедневная плата с каждой нации. При неуплате города исключаются, нация удаляется. |

#### Модификаторы содержания

Содержание можно настроить по-разному в зависимости от размера города/нации:

- **По количеству жителей** — через `upkeepModifier` в `town_level` и `nation_level`.
- **По количеству участков** — `town_plotbased_upkeep: true`. Можно задать минимальные/максимальные суммы.

#### Дополнительные настройки содержания

| Параметр (с версии) | Описание |
| :--- | :--- |
| `economy.daily_taxes.nation_pertown_upkeep` (0.95.0.0) | Взимать содержание с нации за каждый город (цена × количество городов). Может учитывать модификатор `nation_level`. |
| `price_town_overclaimed_upkeep_penalty_plotbased` (0.95.0.0) | Штраф за каждый лишний участок сверх лимита. Сумма задаётся в `economy.daily_taxes.price_town_overclaimed_upkeep_penalty`. |
| `economy.daily_taxes.nation_perplot_upkeep` (0.97.3.0) | Содержание нации на основе общего числа участков во всех городах (цена × количество участков). |

#### Освобождение от содержания

| Способ | Описание |
| :--- | :--- |
| **NPC-мэр** | Города с NPC-мэром (`/ta set mayor ГОРОД npc`) не платят содержание. |
| **Флаг upkeep** | `/ta town ГОРОД toggle upkeep` — отключает содержание для конкретного города. |

## 🏦 Городские и национальные банки (Town and Nation Banks)

| Характеристика | Описание |
| :--- | :--- |
| **Банк города и нации** | Любой житель может пополнить банк, но снимать средства могут только мэр (или ассистенты) и лидер нации. |
| **Источники поступлений** | Налоги с жителей/городов автоматически зачисляются в банк. |
| **Расходы** | Захват участков и аванпостов оплачивается из городского банка. |
| **Защита от укрывательства денег** | Мэры и лидеры могут использовать банк как укрытие. Администратор может ограничить это: установить лимит банка (`town_bank_cap` / `nation_bank_cap`) или запретить вывод (`/ta toggle withdraw`). |
| **Жёсткий лимит** (с 0.82.0.0) | При превышении лимита деньги не зачисляются. Уже превышенные банки не обнуляются. |
| **История транзакций** (с 0.96.6.0) | Книга с историей операций: `/town bankhistory {страниц}`, `/nation bankhistory {страниц}`, админские `/ta town {город} bankhistory {страниц}` и `/ta nation {нация} bankhistory {страниц}`. |
| **Перевод в другой город** (с 0.98.5.0) | `/t deposit {$} {город}` — пополнить банк другого города. Требует право `towny.command.town.deposit.othertown` (не дано по умолчанию). |
| **Административные депозиты/снятия** | `/ta town {город} deposit|withdraw {$}`, `/ta nation {нация} deposit|withdraw {$}`. |
| **Лимит банка на основе участков** (с 0.103.0.0) | Если `economy.banks.town_bank_cap_is_plotbased: true`, лимит банка зависит от количества городских участков. |

## 📉 Банкротство города (Town Bankruptcy)

| Характеристика | Описание |
| :--- | :--- |
| **Включение** | `economy.bankruptcy.enabled: true` в `config.yml`. |
| **Причина** | Баланс банка уходит в минус из-за уплаты содержания и/или национального налога. |
| **Выход из банкротства** | Пополнить банк через `/t deposit {сумма}` для погашения долга. |

| :--- | :--- |
| Приглашать новых жителей | Да |
| Открыть город (open) | Да |
| Захватывать новые участки | Да |
| Строить/ломать в городе | Да |

## 💳 Долговые лимиты (Debt Caps)

| Метод расчёта | Описание |
| :--- | :--- |
| **По умолчанию** | (Стоимость города) + ((Количество участков - 1) × Цена захвата) + (Аванпосты × (Цена аванпоста - Цена захвата)). Примерно столько, сколько город потратил на расширение. Отображается в `/town`. |

### Переопределение долгового лимита

| Настройка | Действие |
| :--- | :--- |
| `economy.bankruptcy.debt_cap.maximum > 0` | Жёсткий максимум долга для всех городов (но не уменьшает уже установленный). |
| `economy.bankruptcy.debt_cap.override > 0` | Фиксированный лимит долга для всех городов, игнорирует остальные расчёты. |
| `debt_cap_uses_town_levels: true` | `debt_cap.override` умножается на `debtCapModifier` из `town_level`. Например: модификатор 3.0, override 1000 → лимит 3000. |

### Альтернативный расчёт долга (с 0.99.3.0)

| Настройка | Описание |
| :--- | :--- |
| `debt_cap_uses_fixed_days: true` | Лимит = ежедневное содержание × `economy.bankruptcy.debt_cap.allowed_days` (по умолчанию 7 дней). |
| **Преимущество** | Позволяет точно предсказать время удаления города-призрака. При использовании с `delete_towns_that_reach_debt_cap` удаление произойдёт через заданное число дней. |

### Правила банкротства

| Параметр | Поведение |
| :--- | :--- |
| `delete_towns_that_reach_debt_cap: true` | Город удаляется при достижении лимита долга. Если `false` — не удаляется, но долг дальше не растёт. |
| `do_bankrupt_towns_pay_nation_tax: true` | Банкрот платит налог нации из долга, пока не достигнет лимита. Рекомендуется `true` при системе завоеваний (Siege War), чтобы захваченные города не выходили из нации, не платя налог. Если `false` — банкрот не платит и покидает нацию. |
| `kick_towns_that_reach_debt_cap: true` | Город, достигший лимита долга, исключается из нации за неуплату налога. |

## 🏚️ Городские руины (Town Ruins)

| Характеристика | Описание |
| :--- | :--- |
| **Что это** | Вместо немедленного удаления города (из-за неуплаты или команды) он переходит в состояние руин. Полное удаление происходит спустя `max_duration_hours` (в часах). |
| **Включение** | `town_ruining.town_ruins.enabled: true` в `config.yml`. |
| **Поведение в руинах** | — Участки остаются захваченными, но не могут быть перезахвачены другими.<br>— Мэром становится NPC.<br>— Город можно гриферить.<br>— Жителям запрещено большинство `/town` и `/plot` команд, кроме `/town`, `/town reclaim`, `/town list`, `/town leave`. |
| **Цель** | Предотвратить мгновенный выход из нации через удаление и пересоздание города (особенно полезно для плагинов осад). |

### Дополнительные настройки руин

| Параметр (версия) | Описание |
| :--- | :--- |
| `town_ruining.town_ruins.town_bank_is_sent_to_nation` (0.99.1.0) | Оставшийся баланс банка города переводится в банк нации (по умолчанию `false`). |
| `town_ruining.town_ruins.do_plots_permissions_change_to_allow_all` (0.100.2.0) | Если `true`, каждый час права на участках постепенно открываются для всех (build, destroy, switch, itemuse). Начинается с последних захваченных участков. Примеры темпа: 500 участков / 72 часа = ~7 участков в час; 36 участков / 72 часа = 1 участок каждые 2 часа. Позволяет игрокам из разных часовых поясов успеть «разграбить» руины. |

## 💬 Чат (Chat)

### Поддержка PlaceHolderAPI

Towny предоставляет плейсхолдеры для PlaceholderAPI, список которых доступен на [этой странице вики](https://github.com/TownyAdvanced/Towny/wiki/PlaceholderAPI).

### TownyChat.jar

Для отображения переменных Towny в чате, а также для работы каналов города и нации необходимо скачать и установить плагин **TownyChat.jar**.

---

### Использование TownyChat с другими чат-плагинами

| Шаг | Действие |
| :--- | :--- |
| 1. | Откройте `Towny ChatConfig.yml` (находится в `\plugins\towny\settings\`). |
| 2. | Найдите `modify_chat.enable` и установите `false`: `modify_chat.enable: 'false'` |
| **Результат** | Глобальные каналы (`GLOBAL`) из `channels.yml` не будут изменяться Towny, оставляя форматирование другому чат-плагину. |

---

### Использование TownyChat без других чат-плагинов

| Шаг | Действие |
| :--- | :--- |
| 1. | Откройте `Towny ChatConfig.yml`. |
| 2. | Найдите `modify_chat.enable` и установите `true`: `modify_chat.enable: 'true'` |
| 3. | Настройте формат чата с помощью параметров, описанных ниже. |

---

### ChatConfig.yml

Основной конфигурационный файл чата Towny — `ChatConfig.yml`, расположенный в `\plugins\towny\settings\`.

#### Форматы чата Towny

Для составления строк `Channel_format` используются следующие компоненты (плейсхолдеры):

```
  {worldname} - Displays the world the player is currently in.
  
  {town} - Displays town name if a member of a town.
  {townformatted} - Displays town name (if a member of a town) using tag_format.town.
  {towntag} - Displays the formated town tag (if a member of a town) using modify_chat.tag_format.town.
  {towntagoverride} - Displays the formated town tag (if a member of a town and present) or falls back to the full name (using modify_chat.tag_format.town).
   
  {nation} - Displays nation name if a member of a nation.
  {nationformatted} - Displays nation name (if a member of a nation) using tag_format.town.
  {nationtag} - Displays the formated nation tag (if a member of a nation) using modify_chat.tag_format.nation.
  {nationtagoverride} - Displays the formated nation tag (if a member of a nation and present) or falls back to the full name (using modify_chat.tag_format.nation).
   
  {townytag} - Displays the formated town/nation tag as specified in modify_chat.tag_format.both.
  {townyformatted} - Displays the formated full town/nation names as specified in modify_chat.tag_format.both.
  {townytagoverride} - Displays the formated town/nation tag (if present) or falls back to the full names (using modify_chat.tag_format.both).
   
  {title} - Towny resident Title
  {surname} - Towny resident surname
  {townynameprefix} - Towny name prefix taken from the townLevel/nationLevels
  {townynamepostfix} - Towny name postfix taken from the townLevel/nationLevels.
  {townyprefix} - Towny resident title, or townynameprefix if no title exists
  {townypostfix} - Towny resident surname, or townynamepostfix if no surname exists
   
  {townycolor} - Towny name colour for nation leader/mayor/resident
  {group} - Players group name pulled from your permissions plugin
  {permprefix} - Permission group prefix
  {permsuffix} - Permission group suffix.
  {permuserprefix} - Permission group prefix
  {permusersuffix} - Permission group suffix.
   
  {playername} - Default player name.
  {modplayername} - Modified player name (use if Towny is over writing some other plugins changes).
  {msg} - The message sent.

  {channelTag} - Defined in the channels entry in Channels.yml
  {msgcolour} - Defined in the channels entry in Channels.yml
```

| Параметр | Описание |
| :--- | :--- |
| `spam_time: 0.5` | Контроль спама сообщений (интервал между сообщениями в секундах). |
| `channel_formats` | Определяет внешний вид каждого чат-канала. |
| `tag_formats` | Определяет, как будут выглядеть теги. |
| `colour` | Настройка цветов, применяемых через `{townycolor}` для мэров, лидеров наций и жителей. |
| `modify_chat` | Позволяет полностью отключить все модификации чата со стороны Towny. |
| `per_world: true` | Включает секцию `worlds:` для изменения каналов на основе миров. |

### 💬 Чат-каналы (Chat Channels)

Настраиваются в файле `Channels.yml` в `\plugins\towny\settings\Channels.yml`. По умолчанию доступно шесть каналов, но администратор может создать любое количество новых. В `Channels.yml` задаются команды для подключения и использования каждого канала.

#### Команды быстрого доступа

| Команда | Канал |
| :--- | :--- |
| `/g` | Общий / глобальный чат (General/Global). |
| `/l` или `/lc` | Локальный чат (Local). |
| `/tc` | Городской чат (Town). |
| `/nc` | Национальный чат (Nation). |
| `/ac` | Чат альянса (Alliance — нация + союзники). |
| `/a` или `/admin` | Административный чат (Admin). |
| `/m` или `/mod` | Модераторский чат (Mod). |

Без текста после команды игрок просто подключается к каналу.

#### Настройки каналов

| Параметр | Описание |
| :--- | :--- |
| **Теги** | Задаются для каждого канала и используются в `ChatConfig.yml` через `{channelTag}`. |
| **Права** | Для каждого канала можно задать ноду доступа. |
| **Радиус слышимости** | `-1` — без ограничений, `0` — только в том же мире, положительное число — радиус в том же мире. |

---

### 🔌 Автоматическое подключение к каналам при входе

Используя info/option/meta-ноды в плагинах прав (GroupManager, PEX, bPermissions), можно автоматически помещать игроков в каналы при заходе на сервер.
Нода добавляется в той же секции, где задаются `prefix` и `suffix`.

| Команда | Описание |
| :--- | :--- |
| `/nation set title {житель} {текст}` | Добавляет префикс (титул) к имени игрока. |
| `/nation set surname {житель} {текст}` | Добавляет суффикс (фамилию) к имени игрока. |

### Команды для мэров городов

| Команда | Описание |
| :--- | :--- |
| `/town set title {житель} {текст}` | Добавляет префикс (титул) к имени жителя. |
| `/town set surname {житель} {текст}` | Добавляет суффикс к имени жителя. |

> Титул/суффикс, выданный мэру, переопределяет `MayorPrefix`/`MayorPostfix` из `townLevels` конфига, но сохраняет цвет имени мэра (по умолчанию голубой).

### Ограничение цветовых кодов

| Действие | Настройка |
| :--- | :--- |
| **Полный запрет** | Изменить `name_remove_regex` в `config.yml`, убрав символ `&`: `'[^\P{M}a-zA-Z0-9._\[\]-]'` |
| **Разрешение через право** | В `config.yml` установить `filters_colour_chat.modify_chat.does_adding_colour_codes_require_permission_node: true`. Тогда лидер нации получит право `towny.command.nation.set.title.colours`, а мэр — `towny.command.town.set.title.colours`. По умолчанию эти права никому не выдаются. |

## 🔐 Ноды прав (Permission Nodes)

Towny имеет встроенную систему прав **TownyPerms**, настраиваемую в файле `towny\settings\townyperms.yml`. Ноды автоматически выдаются игрокам в зависимости от их роли, пока они находятся в мире с включённым Towny.

### Базовые роли (присваиваются автоматически)

| Роль | Кто получает |
| :--- | :--- |
| **Nomad** | Игроки без города. |
| **Resident** | Игроки, состоящие в городе. |
| **Mayor** | Владельцы городов. |
| **King** | Владельцы наций. |

### Дополнительные ранги (назначаются мэрами/лидерами)

| Ранг | Назначение |
| :--- | :--- |
| **Town Assistant** | `/town rank add {игрок} assistant` |
| **Nation Assistant** | `/nation rank add {игрок} assistant` |
| **Прочие** | Любые ранги, созданные админом в `townyperms.yml` (банкир, строитель, приглашающий и т.д.) |

### Просмотр и назначение рангов

| Команда | Описание |
| :--- | :--- |
| `/town ranklist` | Показать ранги в городе. |
| `/town rank add {игрок} {ранг}` | Назначить ранг жителю. |
| `/town rank remove {игрок} {ранг}` | Снять ранг с жителя. |
| `/nation rank add {игрок} {ранг}` | Назначить ранг в нации. |
| `/nation rank remove {игрок} {ранг}` | Снять ранг в нации. |

### Приоритеты и префиксы рангов (с 0.99.1.0)

| Нода | Описание |
| :--- | :--- |
| `towny.rankpriority.#` | Приоритет ранга (чем выше число, тем выше приоритет). |
| `towny.rankprefix.&a` | Префикс, добавляемый к имени игрока с этим рангом. |

> Пример: Assistant (`priority 100`, префикс `&a`), Sheriff (`priority 500`, префикс `&b`). Если у игрока оба ранга, отобразится префикс Sheriff.

### Редактирование townyperms в игре (с 0.97.1.0)

| Команда | Описание |
| :--- | :--- |
| `/ta townyperms ?` | Показать доступные команды управления `townyperms.yml` в игре. |

### Важно

Если вы планируете создать ресурсный мир без приватов, не отключайте Towny полностью (`/tw toggle usingtowny`), а сделайте мир **неприватизируемым** (`/tw toggle claimable`). Так игроки сохранят права из TownyPerms, но не смогут захватывать землю.

Полный список всех нод прав можно найти на [странице Permission Nodes](https://github.com/TownyAdvanced/Towny/wiki/Permission-Nodes).

## 🌐 Мультиязычность (Multi Language)

Towny поддерживает несколько языков. Переводы выполняются сообществом на [Crowdin](https://crowdin.com/project/townyadvanced). Если вашего языка нет — запросите его на Discord или в Issue.

### Как это работает

- Язык сообщений Towny определяется по языку, установленному в клиенте Minecraft игрока.
- Администратор может задать язык по умолчанию в `config.yml` (параметр `language: english`). Он используется для консольных сообщений и для игроков, чей язык не поддерживается.

### Переопределение языковых строк

| Папка | Назначение |
| :--- | :--- |
| `towny\settings\lang\override\global.yml` | Строки из этого файла переопределяют переводы для **всех** языков. Полезно для изменения префикса Towny, цветов статусных экранов и т.д. |
| `towny\settings\lang\override\` | Можно добавить файл конкретного языка (например, `ru-RU.yml`), чтобы переопределить перевод только для него. |

> ⚠️ **Важно:** Изменение файлов в `towny\settings\lang\reference\` **не** влияет на игру. Используйте только папку `override`.

### Перевод команд (с 0.99.6.0)

Вся система справки по командам (`/towny ?`, `/town ?` и т.д.) теперь доступна для перевода.
